# Cost Averaging & Value Rebalancing (CAVR) 전략 공식 (Toss Open API 버전)

이 문서는 프로젝트 **tossinvestCAVR**의 핵심 매매 전략인 **무한매수법(Cost Averaging)**과 **...** 밸류 리밸런싱(Value Rebalancing) 전략의 공식 및 토스증권 API 연동 규칙을 정리한 전략 레퍼런스입니다.

---

## 1. 용어 정의 및 토스 Open API 주문 매핑

| 용어 | 설명 | 토스 Open API 매핑 방식 |
| :---: | :--- | :--- |
| **LOC** | Limit on Close (종가 지정가 매매). 종가가 지정가보다 같거나 낮을(매수)/높을(매도) 때 체결. | **US**: `LIMIT` + `CLS` (마감 지정가) 주문 직접 전송.<br>**KR**: 미지원으로 인해 **15:10 LOC 모사 주문 스케줄러**(`job_kr_loc_simulation`)에 의한 지정가 매수로 대체. |
| **MOC** | Market on Close (종가 시장가 매매). 종가에 시장가로 체결. | 미지원으로 인해 **시장가 주문**(`MARKET` + `DAY`)으로 즉시 제출하여 대체 체결 처리. |
| **⭐(star)%** | LOC 매수/매도 기준% 값. 공식: $\text{목표수익률}(\%) - \left(\frac{T}{2} \times \frac{T_{default}}{a_{default}}\right) \%$ | 무매 전/후반전 분할 매수의 기준점 가격으로 사용. |
| **T (Turns)** | 회차. **공통 원칙**: 소수점 자리수 조정(반올림, 올림, 버림 등)을 하지 않고 계산값 그대로 사용. 공식: $\text{누적 매수액} / \text{1회 매수액}$ | 주식 잔고 평가액 추적을 통해 실시간 턴 수 계산. |
| **a (분할수)** | 기본 40분할 기준 외에 사용자가 직접 설정하는 분할 횟수. | 운용 자금 분할 및 1회 매수금 산정의 분모값. |
| **G (기울기)** | 밸류 리밸런싱에서 목표 밸류($V$)의 증가 속도를 조절하는 기울기 파라미터. (보통 10 ~ 30 범위) | 실력공식의 자산 성장 유도 상수. |
| **Pool** | 전략에 할당된 가용 예수금. | TOSS Open API 잔고 및 가용 매수 자금 조회값 기준. |
| **E (Evaluation)** | 현재 보유 주식의 평가액. | TOSS Open API 보유 자산 조회값 기준. |
| **V (Value)** | 현재 사이클의 목표 밸류 (기대 평가액). | 매 2주 단위 실력공식에 의해 도출되는 목표 자산 가치. |
| **리버스 모드** | 원금 소진 후($T > a-1$) 무한 매도와 쿼터 매수를 병행하는 방어 모드. | 토스 API 미지원인 MOC 매도는 시장가(`MARKET`), 쿼터 매수는 LOC(`LIMIT`+`CLS` 또는 지정가)로 집행. |

## 2. 무한매수법 V2.2 방법론

무한매수법 V2.2는 원금 소진율(진행률)에 따라 전반전과 후반전으로 나누어 매수/매도를 수행하며, 원금 소진 시 쿼터손절 모드로 진입합니다.

### 2.1. 기본 매매 로직

#### 1) 전반전 (원금 소진율 < 50%)
*   **매수**: 매일 1회치 분량을 둘로 나누어 LOC 주문 제출.
    *   **평단 LOC 매수 (0.5회치)**: $\min(\text{평단가}, \text{현재가} + 15\%)$
    *   **큰수 LOC 매수 (0.5회치)**: $\min(\text{평단가} \times (1 + \text{⭐}\%), \text{현재가} + 15\%)$
*   **매도**:
    *   전체 수량의 75%: 평단가 + 10% 지정가 매도 (`LIMIT` + `DAY`)
    *   전체 수량의 25%: 평단가 $\times (1 + \text{⭐}\%)$ LOC 매도 (`LIMIT` + `CLS`)

#### 2) 후반전 (원금 소진율 $\ge$ 50%)
*   **매수**: 매일 1회치 분량 전체를 큰수 LOC 주문으로 제출.
    *   **큰수 LOC 매수 (1.0회치)**: $\min(\text{평단가} \times (1 + \text{⭐}\%), \text{현재가} + 15\%)$
*   **매도**: 전반전과 동일하게 수행.

---

### 2.2. 쿼터손절 모드

40회(또는 지정된 분할수 $a$)를 모두 채웠거나, $T > 39$인 상태에서 **평가 손실률이 -10%를 초과**할 경우 강제 진입합니다.

1.  **손절 실행 (MOC 매도 대체)**: 보유 주식 수량의 25%를 토스 일반 시장가 주문(`MARKET` + `DAY`)으로 즉시 전송하여 예수금을 확보합니다.
2.  **자금 분할**: 확보한 손절금을 기준으로 **10분할**하여 신규 매수를 진행합니다.
3.  **쿼터 매수**: 평단가 - 10% 가격과 현재가 + 15% 중 낮은 가격으로 1회치 **LOC 매수**를 매일 진행합니다.
4.  **쿼터 매도**:
    *   전체 수량의 75%: 평단가 + 10% 지정가 매도
    *   전체 수량의 25%: 평단가 - 10% LOC 매도
5.  **종료 및 회귀**: 10회차 이전에 매도가 체결되면 즉시 일반 모드(후반전)로 회귀합니다.

## 2.3. 무한매수법 V4.0 방법론

V4.0은 자금 소진 전의 **"일반 모드"**와 소진 후의 **"리버스 모드"**로 구성되며, T값과 매수액이 잔금에 따라 유동적으로 변합니다.

### 1) 일반 모드 (자금 소진 전)
*   **T값의 변화**:
    *   매수 시: 1회 매수 시 $+1.0T$, 0.5회 매수 시 $+0.5T$.
    *   매도 시: **쿼터 매도(25%) 발생 시** 직전 $T \times 0.75$.
    *   **지정가 매도(75%) 후 LOC 매수 발생 시**: $(\text{직전}T \times 0.25) + \text{매수분}(1 \text{ or } 0.5)$.
*   **유동적 1회 매수금**: 고정액이 아닌 남은 잔금을 기준으로 매일 재계산.
    *   $\text{1회 매수금} = \frac{\text{Pool}}{a - T}$
*   **매수/매도 로직**:
    *   **전반전 ($T < a/2$)**: 0.5회치 평단 LOC 매수 + 0.5회치 별지점(Star% Price) LOC 매수.
    *   **후반전 ($T \ge a/2$)**: 1.0회치 별지점 LOC 매수.
    *   **매도**: 보유량의 25% 별지점 LOC 매도 + 75% 목표 수익률 지정가 매도.
    *   **자전거래 방지**: 매수 가격 설정 시 별지점에서 **Buy Offset** (한국: -10원, 미국: -0.01달러) 적용.

### 2) 리버스 모드 (소진 후 소생 모드)
*   **발동 조건**: $T > (a - 1)$ 시점 진입. 남은 잔금과 매도금을 함께 운용.
*   **리버스 별지점**: **직전 5거래일 종가의 평균**을 기준 가격으로 사용.
*   **매도 로직 (무한 매도)**:
    *   **첫날**: 전체 수량의 $1 / (a/2)$ 분량을 토스 시장가(`MARKET` + `DAY`)로 강제 매도.
    *   **이후**: 별지점 위에서 **LOC 매도**. 매도 수량은 항상 직전 보유 수량의 $1 / (a/2)$ (내림 처리).
*   **매수 로직 (쿼터 매수)**:
    *   첫날은 매수 없음. 둘째날부터 **남은 잔금의 25%(잔금/4)**를 별지점 아래에서 **LOC 매수**.
*   **리버스 T값 업데이트**:
    *   매도 시: $\text{직전}T \times (1 - \frac{1}{a/2})$
    *   매수 시: $\text{직전}T + (a - \text{직전}T) \times 0.25$.
*   **종료(회귀) 조건**:
    *   종가가 평단가 대비 특정 비율 위로 올라오면 리버스 모드 종료 후 **일반 모드로 복귀**.

## 3. 밸류 리밸런싱 (Value Rebalancing) 방법론

기본 밸류 리밸런싱에 하락장 방어력을 추가한 **"실력공식"**을 적용하여 곡선형 목표 밸류($V$)를 구축합니다.

### 3.1. 실력공식 (Skill Formula)

매 사이클(2주)이 시작될 때 다음 사이클의 목표 밸류($V_2$)를 아래 공식으로 갱신합니다. (사이클 진행 중에는 $V$를 변경하지 않음)

$$V_2 = V_1 + \frac{\text{Pool}}{G} + \frac{E - V_1}{2\sqrt{G}} + \text{적립금}$$

*   $V_1$: 이전 사이클의 목표 밸류
*   $V_2$: 다음 사이클의 목표 밸류
*   $E$: 직전 사이클 종료일의 주식 평가액 (TOSS holdings 기준)
*   $\text{Pool}$: 현재 가용 예수금 (TOSS buying-power 기준)
*   $G$: 기울기 계수 (적립식/거치식은 10, 인출식은 20 추천)
*   **적립금**: 해당 사이클에 추가되는 투자금

## 4. 토스증권 특화 호가 및 운영 설정

### 4.1. [TossBroker] 호가 단위 보정 공식 (`adjust_price_by_tick`)
토스 Open API에 주문을 넣기 전 가격을 유효 호가단위로 자동 올림/내림 처리합니다.

*   **미국 시장 (US)**:
    *   주가가 `$1.00` 이상인 경우: 소수점 2자리 보정 (매수 시 올림, 매도 시 내림).
    *   주가가 `$1.00` 미만인 경우: 소수점 4자리 보정 (매수 시 올림, 매도 시 내림).
*   **한국 시장 (KR)**:
    *   KOSPI/KOSDAQ 공용 틱 사이즈에 맞게 원 단위 정수로 강제 절상/절하 보정.
    *   주가 < 2,000원: 1원 단위
    *   주가 < 5,000원: 5원 단위
    *   주가 < 20,000원: 10원 단위
    *   주가 < 50,000원: 50원 단위
    *   주가 < 200,000원: 100원 단위
    *   주가 < 500,000원: 500원 단위
    *   주가 $\ge$ 500,000원: 1,000원 단위

### 4.2. 장중 15분 정기 동기화 주기
웹소켓의 부재로 인한 체결 지연 문제를 극복하기 위해 아래 스케줄로 REST API를 15분마다 정기 구동하여 체결 내역을 동기화합니다.
*   **한국 시장**: 09:00 ~ 15:00 KST 동안 매 15분마다 (`kr_hourly_sync`)
*   **미국 시장**: 09:00 ~ 17:00 ET 동안 매 15분마다 (`us_hourly_sync`)
